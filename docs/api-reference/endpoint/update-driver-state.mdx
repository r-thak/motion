---
title: 'Update Driver State'
openapi: 'PATCH /v1/routes/{route_id}/driver-state'
---

Updates the driver state for a completed route and re-computes all enrichment data (stress factors, fuel burn, overall stress score). Only the fields you provide are updated — omitted fields keep their current values.

<Note>
  The route must have `status: "complete"`. Processing or failed routes return `400`.
</Note>

## Parameters

| Parameter | Range | Default | Description |
|-----------|-------|---------|-------------|
| `fatigueScore` | 0.0 – 1.0 | 0.0 | Driver fatigue level. Higher values increase stress on complex maneuvers. |
| `attentionScore` | 0.0 – 1.0 | 1.0 | Driver attention level. Lower values increase stress scoring. |
| `stressLevel` | 0.0 – 1.0 | 0.0 | Current stress level. Compounds with fatigue and attention. |

## Example

<CodeGroup>
```bash cURL
curl -X PATCH http://localhost:8000/v1/routes/route_01HYX3MPK5XJQJG0NB0PRSDVWC/driver-state \
  -H 'Content-Type: application/json' \
  -d '{
    "fatigueScore": 0.7,
    "stressLevel": 0.4
  }'
```

```python Python
import httpx

response = httpx.patch(
    "http://localhost:8000/v1/routes/route_01HYX3MPK5XJQJG0NB0PRSDVWC/driver-state",
    json={"fatigueScore": 0.7, "stressLevel": 0.4}
)

print(response.json()["driverState"])
# {"fatigueScore": 0.7, "attentionScore": 1.0, "stressLevel": 0.4}
```
</CodeGroup>

After updating, fetch the re-scored telemetry:

```bash
curl http://localhost:8000/v1/routes/route_01HYX3MPK5XJQJG0NB0PRSDVWC/telemetry
```
